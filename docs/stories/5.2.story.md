# Story 5.2 — Usuários: Alteração de Senha (Self-Service + Admin Reset)

## Status
Ready for Review

## Story
**Como** usuário do MidiaCore,
**Quero** poder alterar minha própria senha e (se admin) resetar a senha de outros usuários,
**Para que** eu possa manter a segurança da minha conta e administrar acessos da equipe.

## Acceptance Criteria

### AC-1: Self-Service — Usuário altera própria senha
- [ ] Deve existir uma opção "Alterar Senha" acessível ao usuário logado (ex: modal no `UserProfileModal` ou página de perfil)
- [ ] Formulário requer: **Senha atual**, **Nova senha**, **Confirmar nova senha**
- [ ] Nova senha deve ter mínimo 8 caracteres
- [ ] As senhas nova e confirmação devem ser iguais (validação client-side)
- [ ] Uso de `supabase.auth.updateUser({ password: newPassword })` — Supabase valida a sessão ativa
- [ ] Sucesso: exibir mensagem "Senha alterada com sucesso" e fechar modal
- [ ] Erro: exibir mensagem de erro clara (ex: "Senha atual incorreta")

### AC-2: Admin Reset — Admin reseta senha de outro usuário
- [ ] Na `UserManagementTable`, admins devem ter botão "Resetar Senha" por usuário
- [ ] Ao clicar, modal com campos: **Nova Senha** + **Confirmar Nova Senha** (admin digita)
- [ ] Validação client-side: mínimo 8 caracteres, senhas iguais
- [ ] Backend: `PATCH /api/users/reset-password` com `{ userId, newPassword }`
- [ ] Handler usa `supabaseAdmin.auth.admin.updateUserById(userId, { password: newPassword })`
- [ ] Sucesso: exibir mensagem "Senha resetada com sucesso"
- [ ] Admin NÃO pode resetar a própria senha por este fluxo (usar AC-1)
